    ECP v1.2: Updated Plane Screening Workflow

    Exfoliating Crystallographic Plane (ECP) v1.2

    This page documents the current production workflow for screening exfoliable crystallographic planes from non-vdW bulk materials. The method has been updated to match the latest implementation used in version_4.0.py.

    What Is New in v1.2

    • Automated XRD generation from crystal structures using pymatgen; no manual VESTA export is required.
    • Candidate hkl processing is more conservative: low-intensity reflections can be filtered, obvious duplicates such as 001/002 and 001/00-1 are merged, but non-equivalent directions such as 102 and 012 are retained until slab generation.
    • Before cleavage scanning, all generated slabs are compared. Only slabs with identical structures are deduplicated; different slab terminations or different internal stackings are kept.
    • Broken bonds are counted with StructureGraph + jimage, avoiding manual image mapping and reducing periodic-boundary ambiguity.
    • Cleavage-plane scanning supports fixed, event, and auto modes. The event mode evaluates only the heights where the bond topology or V_old can change.
    • The main screening metric in the current workflow is bonding_density = N / V_old, while N / A is kept as an auxiliary area-normalized descriptor.

    Updated Workflow

    01

    Auto XRD

    Generate reflection candidates directly from the bulk structure and apply intensity thresholds before slab construction.

    02

    Clean hkl List

    Remove only trivial duplicates up front, while keeping non-equivalent directions until their slab structures are actually compared.

    03

    Build Stable Slabs

    Use the production slab settings with fixed vacuum and the stable internal scan window based on 4 × ceil(d).

    04

    Compare Slabs

    Deduplicate only when two generated slabs are structurally identical, rather than assuming symmetry reduction is always safe.

    05

    Count Broken Bonds

    Track crossing bonds with StructureGraph + jimage so periodic edges are evaluated explicitly and reproducibly.

    06

    Smart Scan

    Choose between fixed-step and event-point scans automatically, then rank candidate cleavage planes with N / V_old.

    1. Generate XRD Candidates Automatically

    XRD tables are generated directly from the bulk structure using XRDCalculator. The 2θ range and minimum intensity threshold are configurable.

    2. Build the Candidate Plane List

    The candidate list is cleaned in two stages. First, trivial duplicates are merged at the hkl level. Second, slab structures are compared after slab generation, and only truly identical slabs are skipped before scanning.

    3. Generate Stable Slabs

    The current slab construction follows the stable production settings:

    • SlabGenerator is used with center_slab = False.
    • min_slab_size = 4 × ceil(d).
    • min_vacuum_size = 5 Å.
    • The internal scan window is taken from zmin + ceil(d) to zmin + 3 × ceil(d).

    4. Count Crossing Bonds with StructureGraph + jimage

    For each scan plane, the slab is converted to a periodic bond graph. A bond is counted as broken only when the two endpoints of the periodic edge lie on opposite sides of the plane. The jimage information is kept explicitly.

    5. Optimize the Scan

    The workflow supports three scan modes:

    • fixed: traditional constant step-size scan.
    • event: scan only the interval midpoints between structural event heights.
    • auto: choose the cheaper one between fixed and event for the current slab.

    6. Rank Cleavage Planes

    For each slab, the best internal cleavage position is selected by minimizing:

    bonding_density = N / V_old

    where N is the number of broken bonds and V_old is the layer-spacing-based geometric volume used in the current implementation. The code also records:

    • bonding_density_area = N / A
    • broken bond pairs with jimage
    • scan mode and scan-plane count
    • slab-level deduplication metadata

    Download Scripts

    • version_4.0.py: the current production script for material screening.
    • prepare_newvasp_jobs.sh: helper script for preparing per-material job folders; modify paths and environment settings according to your own machine.
    • submit_all_newvasp.sh: helper script for submitting all prepared jobs; modify it according to your own cluster usage.

    Note: version_4.0.py is the core materials-analysis script. The job-preparation and submission shell scripts are only templates and should be adjusted to match your own server environment, paths, and scheduler settings.

    Dependencies

    • Python 3.9-3.12
    • numpy, pandas, matplotlib, pymatgen, ase, openpyxl, tqdm

    Scope

    ECP v1.2 is intended for fast screening of potentially exfoliable crystallographic planes before higher-cost first-principles validation. The current implementation emphasizes reproducibility of slab construction, explicit periodic bond handling, and reduced redundant scans.
